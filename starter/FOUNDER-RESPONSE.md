# Founder Response

Hi Thabo,

Thank you for trusting me with the first version of your product. I focused on building a strong foundation that demonstrates the main value of the platform while keeping the project achievable within the sprint.

## Features Completed

- Home page displaying nearby items.
- Search and filtering functionality.
- Item detail page.
- Two-step booking flow ending with confirmation.
- Login screen.
- Responsive design for desktop and mobile.

## Decisions I Made

### Login before browsing

I decided not to force users to log in before viewing items.

Although capturing emails can help growth, forcing authentication before users understand the value of the product creates friction and may reduce engagement. Instead, users can browse first and log in when they are ready to interact with the platform.

### Fake urgency countdown

I chose not to add countdowns such as "3 people are viewing this item."

Because this project uses mock data, displaying fake activity would mislead users and reduce trust. Building long-term trust is more valuable than creating artificial urgency.

### Messaging

I postponed messaging.

A messaging system requires backend infrastructure, user authentication and real-time communication. Since the sprint was limited, I prioritised the core booking journey.

### Maps

Maps were deferred because they require location services and external APIs. Showing nearby items using mock data demonstrates the concept without increasing complexity.

### Offline mode

Offline support was postponed because Progressive Web App functionality requires additional implementation outside the scope of this sprint.

### Ratings, Reviews, Wishlist and Referral Codes

These were intentionally left for future versions. The priority was to deliver a reliable marketplace experience before adding engagement features.

## Summary

My goal was to deliver a stable, responsive MVP that demonstrates the product's core value while avoiding unnecessary complexity. The project is structured so additional features can be added in future iterations.
